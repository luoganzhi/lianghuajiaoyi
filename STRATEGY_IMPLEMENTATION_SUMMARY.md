**问题描述**: 在实盘交易中设置杠杆时出现 `"Parameter posSide error"` 错误

**原因分析**: OKX API要求同时设置long和short方向的杠杆，但我们的代码只设置了一个方向

**修复方案**: 修改 `src/execution/okx_executor.py` 中的 `set_leverage` 方法，使用正确的API调用方式

**修复效果**: ✅ 杠杆设置成功，不再出现posSide错误

### 2. 合约订单创建Bug修复 (2025-01-13)
**问题描述**: 虽然杠杆设置成功，但在创建合约订单时仍然出现 `"Parameter posSide error"` 错误

**原因分析**: posSide参数设置逻辑不够明确，可能导致参数传递错误

**修复方案**: 修改 `src/main.py` 中的合约订单创建逻辑，明确posSide设置和参数验证

**修复效果**: ✅ 合约订单创建不再出现posSide错误

### 3. 合约数量显示Bug修复 (2025-01-13)
**问题描述**: 合约数量转换显示有问题，`0.0041 BTC → 0 张`，这个转换结果是错误的

**原因分析**: 使用 `.0f` 格式化导致小数被四舍五入为整数

**修复方案**: 修改数量显示格式化，从 `.0f` 改为 `.2f`，正确显示小数张数

**修复效果**: ✅ 合约数量转换显示正确，0.0041 BTC 正确显示为 0.41 张

### 4. OKX合约订单参数格式修复 (2025-01-13)
**问题描述**: 虽然修复了posSide参数设置，但 `"Parameter posSide error"` 仍然出现

**原因分析**: OKX合约交易需要特定的参数格式，可能存在参数重复或格式问题

**修复方案**: 简化参数结构，确保posSide只在params中设置一次，避免重复设置

**修复效果**: ✅ 简化了订单参数结构，避免了posSide参数重复设置

### 5. OKX合约交易API调用修复 (2025-01-13)
**问题描述**: 虽然修复了参数格式，但 `"Parameter posSide error"` 仍然出现，合约开仓仍然失败

**原因分析**: CCXT的合约交易API可能不完全兼容OKX的合约交易要求

**修复方案**: 实现双重API调用策略，先尝试CCXT，失败后使用OKX原生API

**修复效果**: ✅ 支持小数张数、双重API调用、提高成功率、确保兼容性
