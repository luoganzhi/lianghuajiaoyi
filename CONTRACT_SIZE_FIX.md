# 合约数量修复总结

## 🔍 问题发现

用户反馈：**"而且开仓数量还是0.01张，是哪里没改到吗？"**

经过检查发现，虽然`calculate_futures_position_size`函数已经正确计算为0.04张，但在`execute_futures_trade`函数中还有一处写死了0.01张。

## 🔧 问题分析

### 1. 计算函数正确
`calculate_futures_position_size`函数计算正确：
- 保证金：1 USDT
- 每张合约价值：0.24 USDT
- 计算公式：1 / 0.24 * 0.01 = 0.0417
- 最终结果：0.04张

### 2. 问题所在
在`execute_futures_trade`函数的第175行，代码写死了`'sz': '0.01'`：

```python
order_params_native = {
    'instId': symbol_for_trade,
    'tdMode': 'isolated',  # 逐仓模式
    'side': side,
    'ordType': order_type,
    'sz': '0.01'  # 写死为0.01张，测试止盈单功能
}
```

## 🔧 修复方案

### 修复前（错误）
```python
'sz': '0.01'  # 写死为0.01张，测试止盈单功能
```

### 修复后（正确）
```python
'sz': str(contract_value_usdt)  # 使用传入的合约数量
```

## 📍 修复位置

### 1. execute_futures_trade函数
**位置**: `src/main.py` 第175行
**修改**: 使用传入的合约数量而不是写死的0.01

### 2. 相关打印信息
**位置**: `src/main.py` 第177-180行
**修改**: 更新打印信息显示实际计算的数量
