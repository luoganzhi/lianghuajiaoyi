# 止盈方向逻辑修复总结

## 🔍 问题发现

用户反馈：**"止盈单我看了一下，方向设置错了，我现在查询的持仓是做空，那么我需要在低价的时候买入平仓"**

经过检查发现，代码中确实存在止盈方向设置错误的问题。

## 🔧 问题分析

### 1. 止盈方向逻辑
**正确的止盈方向**：
- **做多止盈**: 价格上涨时卖出 (sell)
- **做空止盈**: 价格下跌时买入 (buy)

### 2. 函数设计
`set_take_profit_order` 函数的设计是：
```python
# 对于做多，止盈是卖出；对于做空，止盈是买入
tp_side = 'sell' if side == 'buy' else 'buy'
```

**函数期望的输入**：
- 传入开仓方向 (`side`)
- 函数内部自动转换为止盈方向 (`tp_side`)

### 3. 问题所在
代码中有些地方直接传入了止盈方向，而不是开仓方向，导致：
- 传入 `'sell'` → 函数返回 `'buy'` (错误)
- 传入 `'buy'` → 函数返回 `'sell'` (错误)

## 🔧 修复方案

### 修复前（错误）
```python
# 程序启动时设置止盈
if position_type == 'long' or position_type == '':  # 做多
    tp_side = 'sell'  # 直接设置止盈方向
else:  # 做空
    tp_side = 'buy'   # 直接设置止盈方向

tp_order = set_take_profit_order(account, symbol, tp_side, ...)
```

### 修复后（正确）
```python
# 程序启动时设置止盈
if position_type == 'long' or position_type == '':  # 做多
    entry_side = 'buy'  # 设置开仓方向
else:  # 做空
    entry_side = 'sell' # 设置开仓方向

tp_order = set_take_profit_order(account, symbol, entry_side, ...)
```

## 📍 修复位置

### 1. 程序启动时止盈设置
**位置**: `src/main.py` 第690-695行
**修复**: 传入开仓方向而不是止盈方向

### 2. 持仓监控中止盈设置
**位置**: `src/main.py` 第780-789行
**修复**: 传入开仓方向而不是止盈方向

### 3. 做多开仓后止盈设置
**位置**: `src/main.py` 第1075-1082行
**修复**: 传入开仓方向而不是止盈方向

### 4. 做空开仓后止盈设置
**位置**: `src/main.py` 第1170-1176行
**修复**: 传入开仓方向而不是止盈方向
