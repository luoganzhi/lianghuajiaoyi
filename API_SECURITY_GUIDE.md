# 🔐 API密钥安全配置指南

## ⚠️ 安全警告

**重要提醒**: 你的API密钥已经暴露在代码中，请立即采取以下安全措施：

1. **立即重置所有API密钥**
2. **检查是否有未授权的交易**
3. **启用API密钥的IP白名单限制**
4. **设置最小权限**

## 🛡️ 安全配置步骤

### **步骤1: 创建环境变量文件**

```bash
# 复制示例文件
cp .env.example .env

# 编辑环境变量文件
nano .env  # 或使用你喜欢的编辑器
```

### **步骤2: 填入真实API密钥**

在 `.env` 文件中填入你的真实API密钥：

```bash
# 模拟盘API配置
SIM_API_KEY=你的真实模拟盘API密钥
SIM_API_SECRET=你的真实模拟盘API密钥
SIM_API_PASSWORD=你的真实模拟盘API密码

# 实盘API配置
REAL_API_KEY=你的真实实盘API密钥
REAL_API_SECRET=你的真实实盘API密钥
REAL_API_PASSWORD=你的真实实盘API密码

# 代理设置（可选）
PROXY=http://127.0.0.1:7890

# 其他配置
IS_SIMULATED=false
```

### **步骤3: 验证配置**

运行程序验证配置是否正确：

```bash
python src/main.py
```

## 🔒 安全最佳实践

### **1. 环境变量文件保护**

- ✅ `.env` 文件已添加到 `.gitignore`
- ✅ 永远不会提交到Git仓库
- ✅ 每个开发者都有自己的 `.env` 文件

### **2. API密钥管理**

- 🔑 定期更换API密钥
- 🔑 使用最小权限原则
- 🔑 启用IP白名单
- 🔑 监控API使用情况

### **3. 代码安全**

- 📝 不要在代码中硬编码密钥
- 📝 使用环境变量或配置文件
- 📝 定期检查代码中的敏感信息

## 🚨 紧急处理

### **如果密钥已泄露：**

1. **立即登录交易所账户**
2. **删除所有API密钥**
3. **创建新的API密钥**
4. **检查账户活动**
5. **联系交易所客服**

### **防止未来泄露：**

1. **使用环境变量**
2. **定期代码审查**
3. **启用Git钩子检查**
4. **使用密钥管理服务**
